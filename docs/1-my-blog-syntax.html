My blog syntax

I wanted to write my own parser for my blog, but I realized that Markdown is too complicated. This blog discusses the syntax I ended up using to write my blog posts.

Published on 10th Apr 2021. Last modified on 14th Apr 2021.

Let's talk about markdown

Specifically, how to parse Markdown without using regex.
Ever tried that?

I don't think that a proper Markdown-to-HTML converter exists that doesn't use any regex at all. I searched a bit, and the only result I could find was this. A program written in Go, which is a language that I know nothing about.

Let me put down my requirements first. I want a program that -
  • Is open-source
  • Is simple and light-weight
  • Does not use regex
  • Reads plaintext (from file or stdout)
  • Spits out HTML
Also, the format that I shall write the plaintext must be -
  • Readable
    By 'readable' I mean that the raw, plaintext file should be easily readable and understandable. That is, the formatting itself must not hinder while reading the raw file.
  • Familiar
  • Simple
  • Concise
  • Does not need regex


So, that puts Markdown out of the question, as it absolutely requires regex.
Also, pure HTML is too verbose. While reading a plaintext file, two[1] or four[2] extra characters are OK. But seventeen[3]?? Not OK.
